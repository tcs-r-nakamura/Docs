# LLM・AI基礎知識まとめ

---

## 1. LLMとは何か

### LLMの定義
LLM（Large Language Model：大規模言語モデル）とは、膨大なディープラーニングとデータセットによって構築されたAIモデル。自然言語処理に特化したモデルであり、テキストの生成をはじめ、あらゆる場面で活用される。

### 従来のプログラムとの違い
- 従来のプログラム：人間が書いたルールを正確に実行する
- LLM：データからパターンを学び、確率的に応答する

### 何を入力し何を出力するか
- **入力**：テキスト（プロンプト）
- **出力**：テキスト（最も確率が高いトークン列）

内部的には、入力テキストをトークン列に変換し、各トークンの文脈を踏まえて「次に来るトークン」の確率分布を計算し続けることで文章を生成する。

### なぜ自然言語で命令できるのか
人間の書いた膨大な文章から言葉の意味・回答・命令と実行のパターンを統計的に吸収しており、命令→応答のペアで追加学習させることで命令に従う能力を強化しているから。

## 2. トークンとは何か

### トークンの意味
テキストデータを数としてカウントするための単位としてトークンが使用されている。

### 文字数や単語数との違い
文字数や単語数によって使用されるトークン量が違う
英語: 1単語 ≒ 1トークン（例: "hello" → 1トークン）
日本語: 1文字が複数トークンになることが多い（例: "こんにちは" → 約3〜5トークン）

### トークン消費とコストの関係
APIの利用料金はトークン数に比例する。入力トークン数と出力トークン数の合計で課金されることが多く、入力は並列処理できるが、出力は1トークンずつ逐次生成されるため出力の方が単価が高い。
コスト ＝ (入力トークン数 × 入力単価) + (出力トークン数 × 出力単価)

### 長い入力で何が起きるか
・コンテキストウィンドウを超えて古い内容が切り捨てられる。
・中間部の情報を見落としてしまう可能性がある
・レイテンシ・コストの増加
・解答精度の低下

## 3. コンテキストウィンドウとは何か

### コンテキストウィンドウの意味
LLMが一度に処理できるテキスト最大長のこと。このウィンドウサイズによってAIがどれだけの情報を記憶し、考慮に入れながら文章を生成したり、質問に答えたりできるかが決まる。

### 会話の履歴や指示が出力にどう影響するか
以前のやり取りの詳細を忘れずに実行するため、出力に以前の会話も影響する。しかし、履歴が長くなると中間部の情報を見落とす。なので、重要な指示はシステムプロンプトか直近のターンに置く。

### 会話が長いと情報はどう扱われるか
会話が長くなるとコンテキストウィンドウを圧迫する。上限に達した場合、多くのシステムでは古い会話を要約・圧縮したり、切り捨てたりする処理が行われる。その結果、会話の序盤に書いた情報が後半で参照されなくなることがある。これを「Lost in the middle」問題とも呼ぶ。

### 学習とは何が違うのか
コンテキストウィンドウに情報を入れても、モデル自体は「学習」しない。セッションが終われば記憶は消える。学習はモデルのパラメータを更新する処理であり、大量のデータと計算資源が必要で時間もかかる。コンテキストはあくまで「その場限りの参照」にすぎない。

## 4. AIの出力とは何か

### 次トークン予測の方法
LLMの出力は「次に来るトークンを予測する」を繰り返した結果である。

### 温度（Temperature）
温度(Temperature)が「分布の形を決める」こと
モデルは毎回、全トークンに確率を割り当てる。温度はその確率分布を変形させる。低い値に設定すると、AIは最も可能性の高い単語を選びやすくなり、安定した出力になる。高い値に設定すると、ランダムな単語を選択し、多様性のある出力になる。

```
入力："東京は日本の"

Temperature = 1.0    Temperature = 0.2    Temperature = 1.5
「首都」  42%        「首都」  95%        「首都」  25%
「都市」  18%        「都市」   3%        「都市」  22%
「中心」  12%        「その他」  2%       「中心」  18%
「大きな」 8%                             「大きな」17%
「その他」21%                             「その他」18%
```

### サンプリング
サンプリングが確率分布からどう選ぶかである。
サンプリング手法
①Greedy(貪欲法)：最も確率の高いトークンを選択する
②Random Sampling(ランダムサンプリング)：確率に比例してランダムに選択する
③Top-kサンプリング：確率上位k個に候補を絞ってからランダムに選択する
④Top-pサンプリング：確率の高い順に累積し、合計がpを超えるまでの候補から選択する

### 同じ入力でも毎回出力が変わる理由
確率分布からのサンプリングが行われるため。Temperatureが0より大きい限り、毎回異なるトークンが選ばれる可能性がある。

### ハルシネーションが生じる理由
モデルが行っているのは、次に来そうなトークンを選んでいるだけで、その情報が事実かどうかを検証する仕組みを持っていないため、ハルシネーションが生じる
- 内在的ハルシネーション：学習したデータと異なる結果を出力する現象
- 外在的ハルシネーション：存在しないデータを結果として出力する現象

## 5. エージェントとは何か

### 通常のLLMチャットとの違い
**通常のLLMチャット**
入力 → モデル → 出力で完結する。

**エージェント**
目標 → モデル → 行動 → 結果を観察 → （目標が達成されるまでループ）
目標に向けて複数のステップを自律的に計画・実行し、ツール（検索・コード実行・ファイル操作など）を活用しながら結果を出す。ユーザーが次の行動を指示しなくても、自分で判断して動き続ける点が大きな違い。

### コードベース読解の仕組み
エージェントはファイル読み込み・検索・grep等のツールを使い、コードベースを段階的に調査する。「関連ファイルを特定→内容を読む→依存関係を追う」という反復的なプロセスで全体像を把握していく。

### AIエージェントの何が強力なのか
・長時間・多ステップのタスクを自律実行できる
・外部ツール（Web検索・コード実行・API呼び出し）と連携できる
・計画・実行・検証のループを自分で回せる
・人間が介在しなくても複雑な作業を完遂できる

### 自律性が増えることで起こる問題
・意図しない行動：指示の解釈ミスが大きな影響をもたらす
・エラーの連鎖：途中の誤判断が後続ステップに波及する
・説明責任の問題：なぜその行動を取ったか追跡が難しい
・コスト管理の難しさ：ループや過剰なツール呼び出しで費用が膨らむ

## 6. プロンプトエンジニアリング

### 曖昧な入力と具体的な入力の差
曖昧な例：「車の画像」
具体的な例：「赤いスポーツカーが海沿いの道路を夕焼けの中で走っている写真」

具体的な入力は期待通りの出力が得られやすい。曖昧な入力ではAIが不足している情報を補完しようとするが、その補完が意図と一致する保証はない。

### AIにとって「具体的」とは何を意味するか
AIにとって具体的とは、補完を起こさないということ
①誰向けか
②何のためか
③どんな形式か
④どこまでやるか

### 評価基準が曖昧な場合の問題
AIは明示されていない評価基準を自動的に補完しようとする。例えば「良い文章を書いて」という指示では、AIが「良さ」を独自の解釈をおこなってしまい、意図した出力にならない可能性がある。評価基準が曖昧なほど、ハルシネーションや的外れな出力のリスクが高まる。

### AIの出力をどう検証するか
複数回生成：同じプロンプトで複数の出力を比較し一貫性を確認する
ステップ分解：長い出力を段階的に生成させ、各ステップで確認する
Chain-of-Thought：推論過程を出力させることで誤りを発見しやすくする
人間によるレビュー：重要な用途では必ず人間が最終確認する

## 深掘り質問

### 1. LLMとは何か
**Q. 事前学習と追加学習は何が違いますか**

- 事前学習：モデルに基本的な言語能力や語彙、知識を習得させるために教師なし学習を行うこと
- 追加学習：事前学習済みのモデルに対し、「命令→応答」ペアなどの目的別データで再学習させ、特定の能力を強化すること

**Q. なぜ事前学習だけでは「命令に従うAI」にならないのですか**

-事前学習でやっているのは次に来る単語を当てることであるため、命令に従う学習を行っているわけではないから

### 2. トークンとは何か
**Q. なぜ文字数ではなくトークンという単位が必要なのですか**

- 効率よく確率計算するための最適な分割単位、数値に変換する必要がある。その変換の単位がトークン。

**Q. ここでいう「効率のよさ」とは何を指しますか**

- AIがテキストを読む際に、もし全ての単語を辞書のように登録すると、語彙数が膨大になりすぎて計算負荷が高まる。逆に、一文字ずつ処理すると、単語としての意味の繋がりを捉えにくなってしまう。そのため、トークンという単位を用いることで計算負荷と意味の捉えやすさのバランスを取ることができる

**Q. 逆に複数文字が1トークンになる場合はありますか**

- あります。頻繁に使われる文字の組み合わせは、まとめて1トークンとなる
例"hello" → 1トークン

**Q. 文字数が短いのにトークン数が増えやすいケースを1つ挙げてください**

-"<=>"
よく一緒に出る文字列を1トークンにまとめるが、記号は学習時に頻出する文字列の組み合わせとして統合されにくいため、1文字ずつ個別トークンになりやすい。

### 3. コンテキストウィンドウとは何か

**Q. 「Lost in the middle」と、単純なトークン上限超過はどう違いますか**

- Lost in the middle：容量内に収まっているが中間部分への注意が薄れる
- 単純なトークン上限超過:コンテキストウィンドウの容量を超えた

**Q. 同一セッション内でも長い会話で序盤の重要指示が無視されるのはなぜですか**

- コンテキストウィンドウ内に収まっていても、AIは文章全体に均等に注目するわけではないため。(Attention機構)

### 4. AIの出力とは何か

**Q. 「次に来るトークンを予測する」だけでなぜ長文の要約やコード生成ができるのですか**

- 事前学習でインターネット上のウェブサイト、書籍、論文など、膨大なテキストデータ、また、文法や単語の意味、文脈というパターンを大量に学習済みのため、その指示の後に来る「最も自然な続き」が結果として要約やコード生成ができる。 

**Q. Temperatureを0に近づければ常に正確になるのですか**

- 正確になるわけではなく、出力が安定・一貫する。Temperature = 0でも間違う    

**Q. GreedyとTop-pは、出力品質の面でどういうトレードオフがありますか**

- Greedy
- メリット: 出力が安定・再現性が高い
- デメリット: 同じ表現の繰り返しや、不自然にぎこちない文章になりやすい

- Top-p
- メリット: 自然でバリエーションのある文章が生成されやすい 
- デメリット: 確率の低いトークンが選ばれることがあり、出力がブレる場合がある      

**Q. 同じ入力でも出力が変わるのはTemperatureのせいだけですか**

- 他にも要因がある。Temperature以外の主な要因：
- Top-p / Top-k：候補トークンの選び方自体にランダム性がある
- システムプロンプト：APIのバージョンやサービス側の設定が変更されている場合がある

**Q. なぜ人間が見てももっともらしい文章になるのですか**

- 人間が書いた大量のテキストを学習しているため、出力される文章がもっともらしい文章になる。

### 5. エージェントとは何か

**Q. エージェントはなぜコードベース全体を一度に理解せず、段階的に探索するのですか**

- コンテキストウィンドウの容量に上限があるため、全ファイルを一度に読み込むことができないから。

**Q. コンテキストウィンドウが拡充されれば段階的探索をする意味はなくなるのですか**

-無くならない。一度に全て読み込むと重要な情報が抜け落ちてしまったり、関係ない情報まで取得してしまい、出力の精度が低下してしまうため。

### 6. プロンプトエンジニアリング

**Q. 具体性を上げすぎると逆に悪くなる場面はありますか**

- 間違えている具体例を事実として埋め込んだ場合（ハルシネーションを誘導する）
- 前提条件や背景説明を必要以上に詰め込むと、どこが重要かが不明になりAIの精度が低下する

**Q. 推論過程を出させれば常に検証しやすくなるのですか**

- 検証できるのは出力された説明の論理的一貫性であり、AIの内部処理の正しさではない。推論過程が正しく見えても結論が誤っている場合や、逆に推論過程が省略されていても正しい結論が得られる場合がある。

**Q. 「誰向けか、何のためか、どんな形式か、どこまでやるか」を指定すると、なぜ出力品質が上がるのですか**

- 条件を明示することで、AIが参照すべき学習データの範囲が絞られ、より適切な出力が選ばれやすくなるため。
