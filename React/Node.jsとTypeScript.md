# ✅ Node.js とは？

JavaScript をPCやサーバーで動かす実行環境

---

## ✅ ts-node とは？

**TypeScript のコードをそのまま実行できるツール（ランタイム）**です。

普通は、TypeScriptのコードは `tsc`（TypeScriptコンパイラ） を使ってJavaScriptにコンパイル（変換）してから実行します。

でも、`ts-node` を使うと、コンパイルせずに直接 TypeScript ファイルを実行できます。つまり、`node` コマンドの代わりに `ts-node` を使う感じ。

---

## 🔸 型アノテーションとは？

変数・引数・戻り値などに「この値はこの型ですよ」と明示的に指定するもの。

```typescript
let message: string = "Hello";    // ← これが型アノテーション！
```

`: string` の部分が「型アノテーション」。TypeScript に「この変数は文字列型だよ」と伝えている。

---

## ✅ Null 安全性とは？

**「null が入りうるかどうかを、型で明示して制御する」**ということです。

TypeScript における **「オプショナル（optional）」** とは：「そのプロパティや引数が、あってもなくてもOK」という意味の指定です。

---

## インデックスシグネチャ（Index Signature）

TypeScript で「オブジェクトのプロパティ名と値の型が可変だけど、一定のルールで定義されている」ようなケースに使う構文です。

| 用語 | 意味 |
| ---------------  | ------------------------------------------- |
| **インデックスシグネチャ** | 「オブジェクトのキーと値の型を一括で指定する」仕組み |
| **書き方** | `[key: string]: 型` または `[index: number]: 型` |
| **よく使う場面** | データ表、設定、ステータスなど「キーがたくさんあるオブジェクト」 |

🧠 一言で言うと？「プロパティ名が動的に決まる」オブジェクトの型を表す方法。（すべてのプロパティが string 型のキーで、値が number 型、といった指定）

---

## Enum 型（列挙型）

「名前付きの定数の集合」を定義する型。

---

## リテラル型（ユニオン型を使う方法）

「特定の値だけを許容する型」を作るための方法。

| ポイント | Enum型（数値Enum） | 文字列Enum | リテラル型 + ユニオン型 |
| ---------------------| ---------------------------------| -------------------------------------| ---------------------------------|
| 値の型 | 数値（デフォルト0,1,2,...） | 文字列（自分で指定） | 文字列（または数値など） |
| 型安全性 | 弱い（任意の数値が入る） | 強い（指定した文字列以外は不可） | 強い（指定したリテラル以外は不可） |
| 書きやすさ | やや冗長 | やや冗長 | シンプル |
| JavaScript出力コード | 列挙体として出力される | 列挙体として出力される | ただの文字列なので軽量 |
| よく使われるケース | 既存のenumを使いたい場合や互換性 | 型安全重視でEnumの特性も使いたい場合 | 普段使いではこちらが圧倒的に多い |
