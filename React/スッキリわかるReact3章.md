\*\*関数型プログラミング（Functional Programming）\*\*

「関数」を中心にプログラムを組み立てるプログラミングパラダイム（考え方）の一つ



基本的な考え方



関数を「値」として扱う

* 関数を変数に代入できる
* 関数を引数や戻り値として渡せる



副作用を避ける

* 副作用とは、関数の外の状態を変えること



不変性（Immutable）

* データを「変更」しない
* 新しいデータを「生成」する



純粋関数（Pure Function）

* 次の条件を満たす関数です：
* 同じ入力 → 常に同じ出力
* 副作用がない



5\. 関数の合成

* 小さな関数を組み合わせて処理を作る



特長

バグが減る

テストしやすい

並列処理に強い

コードが宣言的で読みやすい



\*\*副作用（side effect）\*\*

関数が「戻り値」以外に、外部の状態に影響を与えること



変数の値を変更する

let count = 0;



function increment() {

&nbsp; count++; // 外部の変数を変更

}



データ構造を直接変更する

function pushItem(arr) {

&nbsp; arr.push(4); // 引数の配列を破壊的に変更

}



入出力（I/O）

console.log("Hello"); // 画面出力

javascript

コードをコピーする

readFile("data.txt"); // ファイル読み込み



データベースやネットワーク操作

saveToDB(user);

sendRequest(api);



時刻や乱数に依存する

Math.random(); // 毎回違う結果

Date.now();    // 呼ぶたびに変わる



副作用あり／なしの比較



❌ 副作用あり：



let total = 0;

function add(x) {

&nbsp; total += x;

}

add は 外部変数 total を直接変更している



✅ 副作用なし：



function add(total, x) {

&nbsp; return total + x;

}

必要な値は すべて引数でもらう



結果は 戻り値として返す



\*\*不変性（Immutability）\*\*

一度作ったデータは「変更しない」という考え方

変更が必要なときは、元のデータを変えずに新しいデータを作る



特長

状態の変化が追いやすい

変更前と変更後が別のオブジェクト

ログ・差分・Undo が簡単

並列処理に強い

同じデータを複数箇所で安全に使える

ロックが不要

関数型プログラミングと相性が良い

純粋関数を作りやすい

副作用を避けやすい

プリミティブ型はもともと不変



❌ 可変（mutable）

let arr = \[1, 2, 3];

arr.push(4);   // 元の配列が変わる（破壊的）



✅ 不変（immutable）

let arr = \[1, 2, 3];

let newArr = \[...arr, 4]; // 新しい配列を作る（非破壊的）
