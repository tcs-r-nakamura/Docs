## 第一級関数とクロージャ

### ✅ 第一級関数（First-Class Functions）とは？

関数を「値」として扱えるということ。つまり関数を：

- 変数に代入できる
- 関数の引数として渡せる
- 関数の戻り値として返せる

### 🔹 構造体ともクラスインスタンスとも異なる、シンプルで柔軟なオブジェクト

一般的な「クラス」や「構造体」のような厳密な型や定義がなくても、オブジェクト（データのまとまり）を自由に作れるということ。

- クラスを定義しなくても、`{}` で簡単にデータと処理（関数）をまとめられる
- 実行中にプロパティを追加したり削除したりもできる

この「柔軟さ」が「シンプルで柔軟なオブジェクト」と言われるゆえん

### 🔹 表現力の高いリテラル記法

プログラムを書くときの「見た目・書き方」がシンプルで、読みやすく書きやすい記法があるという意味。

## 変数宣言

| 宣言方法 | 再代入できる？ | 再宣言できる？ | スコープの特徴 | 使うべき場面 |
| ------- | ------- | ------- | ---------- | -------------------- |
| `const` | ❌ できない | ❌ できない | ブロックスコープ | 値を変更しない変数に最適（基本） |
| `let` | ✅ できる | ❌ できない | ブロックスコープ | 値を後で変えたいときに使う |
| `var` | ✅ できる | ✅ できる | 関数スコープ（古い） | 互換性のためだけに使う（使わない） |

JavaScript なら `const` で宣言したオブジェクトのプロパティは書き換え可能

## 関数型プログラミング

関数型プログラミングとは、参照透過的な関数を組み合わせることで解決するべき問題に対処していく宣言型のプログラミングのスタイルのこと

### 主な特徴

- **副作用がない関数（純粋関数）** を使う
  - 副作用とは：変数を書き換える、外部の状態（ファイルやDB）を変更する、など
  - 純粋関数：同じ入力に対して常に同じ出力を返し、外部の状態に影響を与えない
- 状態を持たない（あるいは極力持たない）
- 変数は再代入せず、「新しい値」を作って処理を進める
- 関数を第一級オブジェクトとして扱う（変数に代入したり、引数として渡したり、戻り値にしたりできる）

JavaScript では、関数に名前をつけずに定義することができます。これが **無名関数（anonymous function）** です。

### 宣言型スタイル

「何をするか」を記述する（命令型：コンピュータに「どうやってやるか」を指示するスタイルと対比）

- 命令型の中に手続き型
- 宣言型の中に関数型

```javascript
// 合計
reduce((a, b) => a + b)

// 最大値
reduce((a, b) => a > b ? a : b)

// 昇順ソート
sort((a, b) => a - b)

// 降順ソート
sort((a, b) => b - a)
```

## コールバック

- 関数を引数として渡す
- 渡した関数が何かの処理が終わった後に「呼び戻される（callbackされる）」
- 短い処理やインラインで使う場合はアロー関数が使われることが多い
- コールバックは通常アロー関数で書く

---

## カリー化（Currying）

複数の引数を取る関数を、1つの引数だけを取る関数の連鎖に変換すること。

```javascript
// 通常の関数
const add = (a, b) => a + b;

// カリー化した関数
const curriedAdd = (a) => (b) => a + b;

console.log(curriedAdd(2)(3)); // 5
```

つまり、`(a, b, c) => ...` という関数を、`a => b => c => ...` の形に変えること。

---

## 部分適用（Partial Application）

関数に必要な引数の一部だけを先に渡して、残りの引数をあとから受け取る新しい関数を作ること。

```javascript
const add = (a, b) => a + b;

// 部分適用で a=2 を固定した関数を作る
const add2 = (b) => add(2, b);

console.log(add2(3)); // 5
```

| 特徴 | カリー化 | 部分適用 |
| ------- | ------------------- | -------------------------------- |
| 目的 | 関数を引数1つずつ受け取る形に変換 | 関数の一部の引数を固定して新しい関数を作る |
| 結果の関数の形 | 引数1つの関数が連鎖的に返される | 元の関数の引数の一部が固定された関数 |
| 実装例 | `(a) => (b) => ...` | `(b) => originalFunc(fixedA, b)` |

| 用語 | 説明 |
| -------- | ---------------------------------- |
| **カリー化** | 複数の引数を取る関数を「1つの引数ずつ取る関数の連鎖」に変換すること |
| **部分適用** | すべての引数を一度に渡さず、**一部だけ先に渡す使い方** |

## クロージャー（Closure）

関数が作られた時の外側の変数（スコープ）を覚えていて、関数の中からその変数にアクセスできる状態のこと

普通、関数の外の変数は関数の外でしか触れません。でもクロージャーは外の関数の中の変数を、返した関数（内側の関数）が「覚えていて使い続けられる」

クロージャーとはカプセル化のようなものである

| 用語 | 意味 |
| --------- | ------------------------------ |
| レキシカルスコープ | ソースコードの構造で決まる変数の有効範囲 |
| クロージャ | レキシカルスコープの仕組みで、関数が外の変数を覚えている状態 |

---

## レキシカルスコープ

「変数の有効範囲（スコープ）が、コードの書かれた場所（構造）によって決まる仕組み」のこと。実行時ではなく、ソースコードの構造（字句レベル＝lexical）でスコープが決まる。

### 特徴

- 関数の定義場所で変数のスコープが決まる
- 関数がどこで呼ばれたか（呼び出し時の状況）には関係しない
- だからクロージャも成立する

## Promise（プロミス）

**「将来完了するかもしれない処理（非同期処理）の結果を表すオブジェクト」**

「まだ終わっていないけど、いつか終わる約束」を表現します。

### なぜ必要？

コールバック関数を使うとネストが深くなりやすい（コールバック地獄）。そこでPromiseを使うと、非同期処理の完了や失敗を簡単に扱える。`.then()` や `.catch()` を使って読みやすく書ける。

| 状態 | 意味 |
| ------------- | -------------- |
| **Pending** | 処理がまだ終わっていない状態 |
| **Fulfilled** | 処理が成功して終わった状態 |
| **Rejected** | 処理が失敗して終わった状態 |

| メソッド | 完了条件 | 成功時の値 | 失敗時 | バージョン |
| ------------ | ------------------------- | -------------------- | ---------------------- | ------- |
| `.all()` | すべてが resolve、またはいずれかが reject | すべての resolve が返した値の配列 | 最初の reject による例外が発生 | ES2015 |
| `.allSettled()` | すべてが完了 | 成果オブジェクトの配列 | 成果オブジェクトの配列を返す | ES2020 |
| `.any()` | いずれかひとつが resolve | 最初の resolve が返した値 | すべてが reject された場合のみ例外が発生 | ES2021 |
| `.race()` | いずれかひとつが完了 | 最初の resolve が返した値 | 最初の reject による例外が発生 | ES2015 |

## プロトタイプメソッド

JavaScript において、オブジェクトの「元（プロトタイプ）」に定義されたメソッドのこと。

あるオブジェクトが直接持っていないメソッドでも、その「親（プロトタイプ）」が持っていれば、そのメソッドを継承して使うことができる。

## スプレッド構文

記号は `...`（ドット3つ）。配列やオブジェクトの中身を展開してコピーしたり、新しい配列・オブジェクトにまとめたりする時に使います。

## 狭義のオブジェクト（純粋なオブジェクト）

「ただのキーと値の集合」だけを持つ、いわゆる「普通のオブジェクト」だけを指します。

```javascript
const person = {
```

## 用語メモ

- **ハンドリング（handling）**：「何かが起きたときに、その状況に対処・処理をすること」
- **スタックトレース（stack trace）**：エラーが発生したときに、「どの関数からどの関数が呼ばれて、どこでエラーが起きたか」を順番に表示したもの

## メソッドチェーン

複数のメソッドを「ドット（`.`）」でつなげて連続して呼び出す書き方です。

```javascript
"hello"
  .toUpperCase()    // "HELLO"
  .split("")        // ["H", "E", "L", "L", "O"]
  .reverse()        // ["O", "L", "L", "E", "H"]
  .join("");        // "OLLEH"
```
