# 【用途と命名対応表】

| 用途 | 命名 |
| --- | --- |
| 単純な取得 | `GetXxx` / `Xxx`（プロパティ） |
| 見つからない可能性あり | `FindXxx` / `TryGetXxx` |
| 外部から取得（API・DB） | `FetchXxx` / `LoadXxx` / `RetrieveXxx` |
| 計算して返す | `CalculateXxx` / `ComputeXxx` |
| 生成して返す | `CreateXxx` / `BuildXxx` / `GenerateXxx` |

---

# 【C#：メソッドの系統と詳細一覧】

## 1. 取得系（Get 系）
- **用途**：値やデータを取得する。軽い取得はプロパティで行う。
- **命名例**：`GetScore()`, `FindPlayer()`, `FetchUserData()`, `LoadSettings()`, `TryGetItem()`
- **注意点**：内部データだけを返す場合は `Get`、外部から取得する場合は `Fetch`/`Load`、検索で存在しない可能性がある場合は `Find`/`TryGet`。

## 2. 設定系（Set 系）
- **用途**：値や状態を設定・代入する。
- **命名例**：`SetVolume()`, `ApplySettings()`, `AssignRole()`, `ConfigureDatabase()`
- **注意点**：単純な代入は `Set`、複雑な初期化や設定は `Apply`/`Configure`。

## 3. 更新系（Update 系）
- **用途**：値や状態を更新・変更する。
- **命名例**：`UpdateScore()`, `RefreshUI()`, `ModifyPlayer()`, `ChangeStatus()`, `IncrementCount()`
- **注意点**：値の変更を伴うことを明確にする。複数要素を更新する場合は `UpdateAll` など。

## 4. 生成系（Create / Build 系）
- **用途**：オブジェクトやデータを新たに生成する。
- **命名例**：`CreatePlayer()`, `BuildReport()`, `GenerateId()`, `InstantiateEnemy()`, `MakeCopy()`
- **注意点**：生成後に副作用がある場合は副作用を名前に明示する。

## 5. 削除系（Delete / Remove 系）
- **用途**：オブジェクトやデータを削除する。
- **命名例**：`DeleteFile()`, `RemovePlayer()`, `ClearCache()`, `DestroyObject()`
- **注意点**：削除対象や範囲が明確になるよう命名。`Clear` は全削除向き。

## 6. 保存・書き込み系（Save / Write 系）
- **用途**：データを永続化、ファイルやDBに書き込む。
- **命名例**：`SaveSettings()`, `WriteLog()`, `StoreData()`, `ExportReport()`
- **注意点**：保存対象と副作用を明確にする。非同期の場合は `XxxAsync` を付与。

## 7. 読み込み系（Load / Read 系）
- **用途**：データを外部から読み込む。
- **命名例**：`LoadConfig()`, `ReadFile()`, `FetchUserData()`, `ImportRecords()`
- **注意点**：`Fetch`/`Load`/`Read` の使い分けを意識。`Fetch` は API 等外部取得向き、`Read` はファイル・ストリーム向き。

## 8. 実行系（Run / Execute 系）
- **用途**：処理・操作を実行する。
- **命名例**：`RunGame()`, `ExecuteCommand()`, `PerformAction()`, `InvokeCallback()`, `StartProcess()`
- **注意点**：処理の範囲を明示。非同期処理は `XxxAsync` を付与。

## 9. 検証系（Validate / Check 系）
- **用途**：条件・値の検証や整合性チェック。
- **命名例**：`ValidateInput()`, `CheckPermission()`, `VerifyData()`, `EnsureInitialized()`, `IsValid()`
- **注意点**：戻り値は `bool` が基本。`Ensure` は例外を投げることを想定。

## 10. 変換系（Convert / To 系）
- **用途**：データ型や形式を変換する。
- **命名例**：`ConvertToJson()`, `ToString()`, `ParseInt()`, `MapToDto()`, `TransformCoordinates()`
- **注意点**：変換内容が明確になるよう命名。例：`MapToDto` は DTO にマッピング。

## 11. 判定系（Is / Has / Can / Should 系）
- **用途**：状態や条件を判定して真偽値を返す。
- **命名例**：`IsActive()`, `HasPermission()`, `CanMove()`, `ShouldUpdate()`
- **注意点**：戻り値は `bool` のみ。命名で真偽値と意味がすぐ分かるようにする。

## 12. イベント・通知系（On / Notify / Raise / Trigger 系）
- **用途**：イベントの発火や通知を行う。
- **命名例**：`OnClick()`, `NotifyUser()`, `RaiseEvent()`, `TriggerAnimation()`
- **注意点**：イベントハンドラは `OnXxx`、通知処理は `Notify`/`Raise`/`Trigger` を使う。

## 13. 初期化系（Init / Setup 系）
- **用途**：オブジェクトや状態を初期化する。
- **命名例**：`InitializeComponents()`, `SetupScene()`, `ResetGame()`, `PrepareData()`
- **注意点**：初期化の範囲を明示。`Reset` は状態を元に戻すことを意図。

## 14. 終了・解放系（Dispose / Close / Stop 系）
- **用途**：リソース解放や停止処理。
- **命名例**：`DisposeResources()`, `CloseConnection()`, `ShutdownServer()`, `ReleaseHandles()`, `StopProcess()`
- **注意点**：`IDisposable` 実装は `Dispose` が必須。副作用がある場合は名前で明示。

## 15. 並列・非同期系（Async 系）
- **用途**：非同期処理、時間がかかる処理。
- **命名例**：`LoadDataAsync()`, `SaveFileAsync()`, `FetchUserDataAsync()`
- **注意点**：.NET では非同期メソッド名に必ず `Async` を付けることが推奨。

---

## 補足

- C# メソッド名は基本「動詞＋目的語」が標準。
- プロパティは軽い取得や状態公開に使い、`Get`/`Set` はなるべくメソッドに限定。
- 命名で処理の副作用や戻り値の意味がすぐ分かることが重要。
