# 【更新系（Update 系）メソッドまとめ】

## 用途

- 値や状態を更新・変更する処理
- 複数の要素を同時に更新する場合や、UI/DBなど副作用を伴う処理に使う

## 命名ルール

- 動詞＋目的語を基本とする
- 値を変更することを明確に名前に反映
- 複数要素を更新する場合は `UpdateAll` などを付与

## 命名例

| メソッド | 説明 |
| --- | --- |
| `UpdateScore()` | スコアを更新 |
| `RefreshUI()` | UI を更新 |
| `ModifyPlayer()` | プレイヤー情報を変更 |
| `ChangeStatus()` | 状態を変更 |
| `IncrementCount()` | カウンタを増加 |
| `UpdateAllSettings()` | 設定を一括更新 |
| `TryUpdateRecord()` | 成功/失敗を返す更新処理（Try パターン） |

## 注意点

1. 更新処理には副作用が伴う場合が多いため、メソッド名で分かるようにする
2. 成功/失敗を呼び出し側で安全に判定したい場合は Try パターンで `bool` を返す
3. DB・ファイル・UI 等の外部要素を更新する場合は `Update` を使い、`Create`/`Load`/`Delete` とは区別する
4. 複雑な更新や副作用が多い場合は、内部処理の順序や例外管理を明確にする
