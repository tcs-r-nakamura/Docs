# 【判定系（Is / Has / Can / Should 系）メソッドまとめ】

## 用途

- オブジェクトや状態の条件を判定して真偽値（`bool`）を返す
- 状態チェック、権限確認、フラグ確認などに使用

## 命名ルール

- 真偽値を返すことが一目で分かるように命名
- 接頭辞は `Is` / `Has` / `Can` / `Should` を使う
  - `Is`：状態や属性の判定
  - `Has`：所有や存在の確認
  - `Can`：実行可能かの判定
  - `Should`：実行すべきかの判定
- 戻り値は必ず `bool`

## 命名例

| メソッド | 説明 |
| --- | --- |
| `IsActive()` | アクティブかどうか |
| `HasPermission()` | 権限を持っているか |
| `CanMove()` | 移動可能か |
| `ShouldUpdate()` | 更新すべきか |
| `IsProcessRunning()` | プロセスが動作中か |
| `HasBackupFiles()` | バックアップファイルが存在するか |
| `CanExecuteUpdate()` | 更新を実行可能か |

## 注意点

1. メソッド名から返り値が `bool` であることが明確に分かる
2. 副作用は原則避ける（判定のみ）
3. 複雑な条件判定の場合は名前に意味を含めて可読性を高める
4. `Ensure` 系は例外を投げる場合に使用し、`bool` 判定とは区別する
