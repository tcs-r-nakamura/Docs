# 【保存・書き込み系（Save / Write 系）メソッドまとめ】

## 用途

- データを永続化する（ファイルやDBへの書き込み）
- ログや設定、レポートなどの保存処理に使用

## 命名ルール

- 動詞＋目的語で命名
- 保存対象や副作用が明確になるようにする
- 非同期処理の場合は `XxxAsync` を付与
- 成功/失敗を返す場合は Try パターン（`bool` 型）を採用

## 命名例

| メソッド | 説明 |
| --- | --- |
| `SaveSettings()` | 設定を保存する |
| `WriteLog()` | ログを書き込む |
| `StoreData()` | データを保存する |
| `ExportReport()` | レポートを出力する |
| `TrySaveFile()` | ファイル保存を試み、成功/失敗を返す |
| `SaveUserPreferences()` | ユーザー設定を保存する |

## 注意点

1. 保存対象や影響範囲を名前で明示する（例：`SaveSettings` は設定を保存することが分かる）
2. 非同期の場合は `XxxAsync` を付ける（例：`SaveSettingsAsync`）
3. 副作用がある処理のため、戻り値や例外管理を明確にする
4. 成功/失敗を呼び出し側で判断したい場合は Try パターンを使用する
