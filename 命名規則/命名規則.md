# 命名規則

## ✅ 真偽値を返す関数・変数の命名（is以外も含む）

「`is`」で始まる命名規則は、真偽値（ブール値）を返す関数の命名に用いられます。

- `is`：`isEmpty`, `isEnabled`, `isAvailable`
- `has`：`hasPermission`, `hasItems`, `hasError`
- `can`：`canSave`, `canEdit`, `canDelete`
- `should`：`shouldRetry`, `shouldShowWarning`

これらは、読み手に「この関数は何かを確認するものだ」と直感的に理解させるのに役立ちます。

---

## 🔵 `is`：状態・条件が「〜であるか？」を表す

意味：何かがある状態かどうか、条件を満たしているかを判断する。返り値：`true` または `false`

| 例 | 意味 |
| --- | --- |
| `isEmpty` | 空であるか？（例：配列や文字列が空） |
| `isEnabled` | 有効化されているか？（ボタンや機能など） |
| `isAvailable` | 利用可能か？（リソースやユーザー名など） |

## 🟢 `has`：何かを「持っているか？」を表す

意味：所有や存在の有無を判断する。返り値：`true` または `false`

| 例 | 意味 |
| --- | --- |
| `hasPermission` | 権限があるか？（ユーザーなど） |
| `hasItems` | アイテムを持っているか？（リストやカートなど） |
| `hasError` | エラーが存在するか？（状態やレスポンスなど） |

## 🟡 `can`：能力や可能性「〜できるか？」を表す

意味：行動や操作が可能かどうかを判断する。返り値：`true` または `false`

| 例 | 意味 |
| --- | --- |
| `canSave` | 保存できるか？（状態が保存可能か） |
| `canEdit` | 編集できるか？（ユーザー権限や状態に基づく） |
| `canDelete` | 削除できるか？（対象や条件による） |

## 🟠 `should`：期待や推奨「〜すべきか？」を表す

意味：何かを実行するべきか、推奨されるかどうかを判断する。返り値：`true` または `false`

| 例 | 意味 |
| --- | --- |
| `shouldRetry` | 再試行すべきか？（エラー発生後など） |
| `shouldShowWarn` | 警告を表示すべきか？（条件に応じて） |

## ✅ よく使われる命名プレフィックス一覧

| プレフィックス | 意味・目的 | 例 | 備考 |
| --- | --- | --- | --- |
| `get` | 値を取得する | `getUser`, `getName` | 値を返す関数・メソッド |
| `set` | 値を設定する | `setName`, `setOptions` | ミューテーター的な役割 |
| `fetch` | データを外部から取得（非同期・API） | `fetchData`, `fetchUser` | ネットワーク処理・Promiseが多い |
| `load` | データやファイルを読み込む | `loadImage`, `loadSettings` | 初期化・UI描画でよく使う |
| `create` | 新しいオブジェクト・構造を作る | `createUser`, `createToken` | DB・ファイルなどで使うことも |
| `build` | 組み立て処理（構造やオブジェクト） | `buildQuery`, `buildComponent` | 構成を定義する処理など |
| `update` | 既存データの更新 | `updateProfile`, `updateConfig` | 差分反映など |
| `delete` | データやファイルの削除 | `deleteAccount`, `deleteItem` | `remove` より強い意味合い |
| `remove` | 特定の要素を取り除く | `removeItem`, `removeChild` | UIや配列などに使われやすい |
| `reset` | 初期状態に戻す | `resetForm`, `resetPassword` | 状態リセット |
| `toggle` | 状態を切り替える | `toggleTheme`, `toggleMenu` | `on ⇄ off` 的な処理に最適 |
| `calculate` | 数値や結果を計算する | `calculateTax`, `calculateScore` | ロジック系処理で |
| `render` | 表示・描画処理をする | `renderPage`, `renderComponent` | ReactやUI系で頻出 |
| `handle` | イベントや例外の処理 | `handleClick`, `handleError` | UIイベントや例外処理 |
| `process` | 一連の処理を実行 | `processData`, `processOrder` | 多段階処理に使いやすい |
| `apply` | 影響を反映・適用 | `applyTheme`, `applyDiscount` | 状態やルールの適用 |
| `convert` | 形式や型の変換 | `convertToJSON`, `convertCurrency` | フォーマット変更など |
| `transform` | 構造や内容の変形 | `transformData`, `transformInput` | データマッピングなどに使う |
| `init` / `initialize` | 初期化 | `initApp`, `initializeStore` | 最初に一度だけ行う処理 |

## 🧠 命名で意識すべきこと

- **動詞 + 名詞の形が基本**（例：`getUserData`, `updateProfileInfo`）
- **副作用の有無が伝わる命名を**
  - `get...` は副作用なし、`set...`, `update...` は副作用あり
- **プロジェクト全体で統一することが重要**

| プレフィックス | 副作用 | 意味・使い方の目安 |
| --- | --- | --- |
| `get` / `is` / `has` / `calculate` / `to` / `format` | ❌ なし | データ取得や変換系 |
| `set` / `update` / `delete` / `save` / `fetch` / `send` / `toggle` / `render` | ✅ あり | 状態変更や出力・通信など |

## ✅ プレフィックス（prefix）とは？

単語の先頭につける語句や文字列のこと。プログラミングでは、関数名や変数名の最初につける「意味を持つ接頭語」のことを指します。

| 用語 | 説明 | 例 |
| --- | --- | --- |
| **Prefix（プレフィックス）** | 接頭辞（名前の先頭につける） | `isReady`, `getUser` |
| **Suffix（サフィックス）** | 接尾辞（名前の末尾につける） | `fileNameTxt`, `userListArray` |
| **Infix（インフィックス）** | 中間に挿入する | `snake_case` の `_` など（まれに使う） |
