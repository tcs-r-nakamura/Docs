# 【実行系（Run / Execute 系）メソッドまとめ】

## 用途

- 処理・操作を実行するメソッド
- 主に副作用がある処理（処理の実行やアクションの開始など）に使用
- UI操作、プロセス起動、コマンド実行などに適用

## 命名ルール

- 動詞＋目的語で命名
- 非同期処理の場合は必ず `Async` を付与
- 処理内容や範囲をメソッド名で明確にする

## 命名例

| メソッド | 説明 |
| --- | --- |
| `RunGame()` | ゲームを開始 |
| `ExecuteCommand()` | コマンドを実行 |
| `PerformAction()` | 指定アクションを実行 |
| `InvokeCallback()` | コールバックを呼び出す |
| `StartProcess()` | プロセスを起動 |
| `TryRunUpdate()` | 成功/失敗を返す実行処理（Try パターン） |
| `RunBackupProcedure()` | バックアップ処理を実行 |

## 注意点

1. 副作用のある処理を実行することが明確になるように命名
2. 戻り値は `void` でもよいが、成功/失敗を返す場合は `bool` 型で Try パターンを採用
3. 非同期処理は `XxxAsync` を付けて、呼び出し側で `await` が使えるようにする
4. 実行範囲や影響範囲が分かる名前にする（例：`RunBackupProcedure` は何を実行するか明示）
