# 【読み込み系（Load / Read 系）メソッドまとめ】

## 用途

- データを外部から読み込む
- ファイル、DB、API など

## 命名ルール

- 動詞＋目的語で命名
- `Fetch` / `Load` / `Read` の使い分けに注意
  - `Fetch`：外部取得向き（API など）
  - `Read`：ファイル・ストリーム向き
  - `Load`：データ全体の読み込み

## 命名例

| メソッド | 説明 |
| --- | --- |
| `LoadConfig()` | 設定を読み込む |
| `ReadFile()` | ファイルを読み込む |
| `FetchUserData()` | ユーザデータを取得する |
| `ImportRecords()` | レコードをインポートする |

## 注意点

1. 読み込み処理は副作用を持たないことが基本
2. 大量データの読み込みや非同期処理の場合は `XxxAsync` を検討
3. `Load` / `Fetch` / `Read` の使い分けで意味を明確にする
