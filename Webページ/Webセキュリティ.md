# Webページ作成におけるセキュリティ注意事項

---

## 1. XSS（クロスサイトスクリプティング）

ユーザーが入力した文字列をそのままHTMLに出力すると、悪意あるスクリプトが実行される。

**対策：**
- ユーザー入力を必ずエスケープする（`<` → `&lt;` など）
- `innerHTML` の使用を避け、`textContent` を使う
- CSP（Content Security Policy）を設定する

```html
<!-- ❌ 危険 -->
element.innerHTML = userInput;

<!-- ✅ 安全 -->
element.textContent = userInput;
```

---

## 2. SQLインジェクション

ユーザー入力をそのままSQL文に埋め込むと、データベースを不正操作される。

**対策：**
- プレースホルダー（パラメータ化クエリ）を使用する
- ORMを活用する

```csharp
// ❌ 危険
string sql = "SELECT * FROM users WHERE name = '" + userInput + "'";

// ✅ 安全
string sql = "SELECT * FROM users WHERE name = @name";
command.Parameters.AddWithValue("@name", userInput);
```

---

## 3. CSRF（クロスサイトリクエストフォージェリ）

ログイン済みユーザーに、意図しないリクエストを送らせる攻撃。

**対策：**
- CSRFトークンをフォームに埋め込む
- `SameSite` Cookie属性を設定する
- 重要な操作はPOSTメソッドを使う

---

## 4. 認証・セッション管理

**対策：**
- パスワードは必ずハッシュ化して保存する（bcryptなど）
- セッションIDは推測困難なランダム値にする
- ログイン後はセッションIDを再生成する
- セッションの有効期限を設ける
- HTTPSを必ず使用する

---

## 5. 機密情報の管理

**対策：**
- APIキーやパスワードをソースコードに直書きしない
- 環境変数や設定ファイル（`.env`）で管理し、Gitに含めない
- `.gitignore` に `.env` を追加する

```
# .gitignore
.env
appsettings.local.json
```

---

## 6. HTTPS（通信の暗号化）

HTTP通信は盗聴・改ざんされる可能性がある。

**対策：**
- 本番環境では必ずHTTPSを使用する
- HTTPアクセスはHTTPSへリダイレクトする
- HSTS（HTTP Strict Transport Security）を設定する

---

## 7. 入力値の検証（バリデーション）

クライアント側のバリデーションは容易に回避される。

**対策：**
- **サーバーサイドでも必ずバリデーションを行う**
- 入力値の型・長さ・形式を厳密にチェックする
- ファイルアップロードは拡張子とMIMEタイプを検証する

---

## 8. エラーメッセージの管理

詳細なエラーメッセージはシステム内部の情報を攻撃者に与える。

**対策：**
- ユーザーには汎用的なエラーメッセージのみ表示する
- 詳細なエラーはサーバーのログにのみ記録する
- 本番環境でデバッグ情報を表示しない

---

## 9. 依存ライブラリの管理

古いライブラリには既知の脆弱性が含まれることがある。

**対策：**
- ライブラリを定期的にアップデートする
- `npm audit` や `dotnet list package --vulnerable` で脆弱性を確認する

---

## 10. クリックジャッキング

透明なiframeを重ねて、ユーザーに意図しないクリックをさせる攻撃。

**対策：**
- `X-Frame-Options: DENY` ヘッダーを設定する
- CSPの `frame-ancestors` ディレクティブを使用する

---

## まとめ

| 脅威 | 主な対策 |
| --- | --- |
| XSS | エスケープ処理・CSP設定 |
| SQLインジェクション | パラメータ化クエリ |
| CSRF | CSRFトークン・SameSite Cookie |
| 認証・セッション | パスワードハッシュ化・HTTPS |
| 機密情報漏洩 | 環境変数管理・.gitignore |
| 通信の盗聴 | HTTPS・HSTS |
| 入力値攻撃 | サーバーサイドバリデーション |
| 情報漏洩 | エラーメッセージの制限 |
| 脆弱なライブラリ | 定期的なアップデート |
| クリックジャッキング | X-Frame-Options設定 |
