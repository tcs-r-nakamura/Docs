# コーディング知識

## try-catch

- `catch` はエラーだけを返す
- 上位で `try-catch` する場合は下位の方で個別の `Exception` を設定（tryの中で起こるだろう例外）し、上位で専用の `catch` を行い、エラー原因特定を容易にする
- `Exception ex` の内容が上位に移行しない場合は書かない
- 上位か下位で `try-catch` を管理する。中間層では管理してはならない

## C# 命名規則

| 種類 | 推奨命名スタイル | 例 |
| ------------------------------ | ----------------------------------------- | -------------------------------------- |
| **クラス / 構造体 / 列挙体 / インターフェース** | **PascalCase** | `HandyAppUpdater`, `IUpdater` |
| **メソッド** | **PascalCase** | `RunHandyApp()`, `TryUpdate()` |
| **プロパティ** | **PascalCase** | `CurrentVersion`, `DownloadFolderPath` |
| **イベント** | **PascalCase** | `UpdateCompleted`, `ErrorOccurred` |
| **フィールド（private）** | **camelCase または _camelCase**（プライベートフィールド） | `_serverAppPath`, `downloadFolderPath` |
| **定数（readonly や const）** | **PascalCase** | `MaxRetryCount`, `DefaultTimeout` |
| **ローカル変数 / パラメータ** | **camelCase** | `filePath`, `result` |
| **名前空間** | **PascalCase** | `ProdCtrlSysUpdater.Commons` |

## SQL に関するメモ

- SQLは定数じゃない（共通的に扱えないから）
- SQLを使用するクラスにSQLフィールドを書く
- 差し戻しの段階では、エラーは起きてはいけない。そこでエラーが起きたら毎回エラーになってしまうため事前にチェックする
