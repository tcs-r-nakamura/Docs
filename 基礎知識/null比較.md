# null の比較は

- ✖ `==`
- ● `is` / `is not`

| 比較方法 | 安全性 | 読みやすさ | 推奨度 |
| ------------- | --- | ----- | -------- |
| `obj == null` | △ | 普通 | △（注意が必要） |
| `obj is null` | ◎ | 高い | ◎（推奨） |

`==` をオーバーロードすると、null比較の挙動も変わる可能性があります。なので、`is null` のような安全な比較方法が推奨される。

---

## オーバーロードとは？

「同じ名前だけど、使い方（引数など）を変えて複数の関数や演算子を作れる仕組み」

C# のオーバーロードもこれと同じで、関数名は同じでも、渡すもの（引数）が違えば別の処理が選ばれます。
