# 非同期処理（async / await）

---

## 1. なぜ非同期処理が必要か

ファイル読み込み・HTTP通信・DB問い合わせなど「待ち時間が発生する処理」を同期で書くと、完了するまでスレッドがブロックされアプリがフリーズする。

```
同期処理：[処理A]→[待ち中…]→[処理B]  ← 待ち中は何もできない
非同期処理：[処理A]→[処理B]→…→[待ち完了後、続きを実行]  ← 待ち中に別の処理が動く
```

---

## 2. 基本構文

```csharp
// async を付けたメソッドは非同期メソッド
// 戻り値は Task<T> または Task（void の代わり）
public async Task<string> GetDataAsync()
{
    // await を付けると、処理が完了するまで「待つが、スレッドは解放する」
    string result = await FetchFromApiAsync();
    return result;
}
```

### 呼び出し側

```csharp
// await を付けて呼び出す
var data = await GetDataAsync();
Console.WriteLine(data);
```

---

## 3. Task と Task\<T\>

| 型 | 説明 |
| --- | --- |
| `Task` | 戻り値なし（void の代わり） |
| `Task<T>` | 戻り値あり（T 型の値を返す） |
| `ValueTask<T>` | 高頻度呼び出し向けの軽量版 Task |

```csharp
// 戻り値なし
public async Task SaveAsync(string data)
{
    await File.WriteAllTextAsync("output.txt", data);
}

// 戻り値あり
public async Task<int> CountLinesAsync(string path)
{
    var lines = await File.ReadAllLinesAsync(path);
    return lines.Length;
}
```

## 4. よくある間違い

### async void は使わない

```csharp
// ❌ NG：例外がキャッチできず、プログラムがクラッシュする
public async void LoadData()
{
    await FetchAsync();
}

// ✅ OK
public async Task LoadDataAsync()
{
    await FetchAsync();
}
```

> **例外：** WPF や WinForms のイベントハンドラだけは `async void` が許容される。

### await を付け忘れる

```csharp
// ❌ NG：Task を返すだけで、完了を待たない
var result = GetDataAsync();  // Task<string> が返るだけ

// ✅ OK
var result = await GetDataAsync();
```

### .Result / .Wait() でデッドロック

```csharp
// ❌ NG（UI スレッドでは特に危険）
var result = GetDataAsync().Result;

// ✅ OK
var result = await GetDataAsync();
```

## 5. 複数の非同期処理を並列実行

```csharp
// 順番に実行（合計 3 秒かかる）
var a = await GetAAsync();  // 1秒
var b = await GetBAsync();  // 2秒

// 並列実行（最大 2 秒で済む）
var taskA = GetAAsync();
var taskB = GetBAsync();
await Task.WhenAll(taskA, taskB);
var a = taskA.Result;
var b = taskB.Result;
```

### Task.WhenAny：どれか1つ完了したら続ける

```csharp
var completed = await Task.WhenAny(taskA, taskB, taskC);
Console.WriteLine("最初に完了したタスクが終わった");
```

## 6. CancellationToken：キャンセル処理

長い処理をユーザーが途中でキャンセルできるようにする。

```csharp
public async Task LongOperationAsync(CancellationToken token)
{
    for (int i = 0; i < 100; i++)
    {
        // キャンセル要求があれば OperationCanceledException をスロー
        token.ThrowIfCancellationRequested();

        await Task.Delay(100, token);
    }
}

// 呼び出し側
var cts = new CancellationTokenSource();

// 5秒後に自動キャンセル
cts.CancelAfter(TimeSpan.FromSeconds(5));

try
{
    await LongOperationAsync(cts.Token);
}
catch (OperationCanceledException)
{
    Console.WriteLine("キャンセルされました");
}
```

## 7. ConfigureAwait

ライブラリコード（UIを持たないコード）では `ConfigureAwait(false)` を付けてデッドロックを防ぐ。

```csharp
// UI 層（ViewModel など）：ConfigureAwait(false) は不要
var data = await GetDataAsync();

// ライブラリ・サービス層：ConfigureAwait(false) を付ける
var data = await GetDataAsync().ConfigureAwait(false);
```

---

## 8. 非同期のまとめ

| ルール | 理由 |
| --- | --- |
| `async` メソッドは `Async` サフィックスを付ける | 呼び出し側が非同期と分かる |
| `async void` は使わない | 例外をキャッチできない |
| `.Result` `.Wait()` は使わない | デッドロックが起きる |
| 複数の独立した処理は `Task.WhenAll` で並列化 | パフォーマンス向上 |
| `CancellationToken` を引数で受け取る | キャンセルに対応できる |
