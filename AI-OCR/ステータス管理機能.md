# ResultView 確認ステータス管理機能 設計書

**機能名**: OCR結果一覧 確認ステータス管理
**対象画面**: ResultView

---

## 1. 背景・目的

OCR結果の運用フローにおいて、各 OCR 結果がオペレーターにより「確認済みか否か」「修正が行われたか」を把握できないという課題がある。本機能は OCR 結果ごとに確認ステータスを記録・表示・絞り込みできるようにし、確認作業の進捗管理と品質追跡を可能にする。

---

## 2. 現状確認

### 2-1. ResultView 構造

| ペイン | 幅 | 内容 |
|--------|-----|------|
| 左 | 250px | ジョブ履歴リスト + 検索・フィルタ |
| 中央 | * | 画像ビューア（バウンディングボックス付き） |
| 右 | 350px | テキストパネル（編集・Undo/Redo） |

### 2-2. 既存フィルタ

| フィルタ | 種別 | バインディング |
|---------|------|--------------|
| テキスト検索 | TextBox | `SearchQuery` |
| ジョブステータス | ComboBox | `StatusFilter` |
| 投入日 | DatePicker × 2 | `DateFrom` / `DateTo` |

フィルタは `FilteredJobHistory`（LINQ / in-memory）で即時反映される。

### 2-3. 既存ステータス体系との区別

現在 `JobStatus`（`OcrViewer.Sdk/Domain/ValueObjects/JobStatus.cs`）が OCR パイプラインの処理状態（`Queued` / `Running` / `Succeeded` / `Failed` / `Canceled`）を管理している。本機能で追加する「確認ステータス」はこれとは独立した概念であり、OCR 処理が完了（`Succeeded`）した後のオペレーター確認フェーズを表す。混同を避けるため型名は `ReviewStatus` とし、UI でも「ジョブステータス」と「確認ステータス」を明確に区別する。

### 2-4. 永続化の仕組み

- ジョブ履歴は `%LOCALAPPDATA%\OcrViewer\job-history.json` にローカル保存
- `JobHistoryRepository` 内の `JobHistoryItemDto` を経由してシリアライズ
- `JobHistoryItem` レコードに `ReviewStatus` プロパティを追加し、同一ファイルに永続化する
- 既存エントリはフィールド欠落時に「未確認」として読み込む（後方互換）

> **補足**: 今回の変更はクライアント側のみ。サーバー側DBへの変更は一切なし。

### 3-2. 画面項目定義

| 項目名 | 種別 | バインディング | 初期値 | 役割 |
|--------|------|--------------|--------|------|
| 確認ステータスバッジ（一覧行） | Border + TextBlock | `JobHistoryItemViewModel.ReviewStatusKey`（DataTrigger 色切り替え）/ `ReviewStatusDisplay`（ラベル文字列） | 「未確認」 | 各ジョブの確認状態をラベル表示 |
| 確認ステータス絞り込み | ComboBox | `ResultViewModel.ReviewStatusFilter` | null（全件） | 確認ステータスによる一覧絞り込み |
| 確認完了ボタン | Button | `ResultViewModel.CompleteReviewCommand` | 非活性（未選択時） | 押下時点の編集有無でステータスを確定 |
| 確認完了ボタン ラベル | TextBlock | `ResultViewModel.CompleteReviewButtonLabel` | — | 編集あり → 「確認完了（修正あり）」、なし → 「確認完了（修正なし）」 |

**確認ステータスバッジ カラーマッピング**

| 値 | 表示文字列 | バッジ色 |
|----|-----------|---------|
| `Unreviewed` | 未確認 | グレー `#E9ECEF` |
| `CompletedWithEdit` | 修正完了 | 緑 `#D4EDDA` |
| `Completed` | 完了 | 青 `#CCE5FF` |

`ReviewStatusDisplay` は上記の日本語文字列を返す（表示専用）。`ReviewStatusFilter` の ComboBox Tag 値は英語小文字（`"unreviewed"` / `"completedwithedit"` / `"completed"`）とし、`FilteredJobHistory` 内で `j.ReviewStatusKey == ReviewStatusFilter` で比較する。DataTrigger によるバッジ色切り替えも同じ `ReviewStatusKey` を使用する。

### 3-3. ユーザー操作と挙動

| 操作 | 挙動 |
|------|------|
| ジョブ履歴から項目を選択 | Right Pane に OCR 結果を表示 |
| テキストを編集しない状態で「確認完了（修正なし）」をクリック | `ReviewStatus = Completed` に更新し永続化。バッジを即時更新 |
| テキストを編集した状態で「確認完了（修正あり）」をクリック | `ReviewStatus = CompletedWithEdit` に更新し永続化。バッジを即時更新 |
| 確認完了後に再度ボタンをクリック | 再確定可（作業ミス訂正のため、完了 ↔ 修正完了 の変更を許容） |
| 確認ステータス ComboBox で値を選択 | `FilteredJobHistory` を再評価し一覧を絞り込み（他フィルタと AND 結合） |
| 確認ステータス ComboBox で「すべて」を選択 | 確認ステータスフィルタを解除 |

### 3-4. ボタン活性制御

| 条件 | ボタン状態 |
|------|-----------|
| ジョブ未選択 | 非活性 |
| 選択ジョブの `JobStatus` が `Succeeded` 以外 | 非活性（OCR 未完了は確認対象外） |
| 選択ジョブの `JobStatus` が `Succeeded` | 活性 |

## 4. 機能仕様

### 4-1. 確認ステータス定義

| 値 | 意味 |
|----|------|
| `Unreviewed`（未確認） | 初期状態（新規ジョブ登録時に自動付与） |
| `CompletedWithEdit`（修正完了） | OCR 結果を編集して確認済み |
| `Completed`（完了） | OCR 結果を編集せず確認済み |

### 4-2. 編集有無の判定

確認完了ボタン押下時点の `_hasEditedDuringReview || TextEditing.HasAnyChanges` で振り分ける。全ページ横断で編集有無を追跡するため、ページ移動のたびに `OnPageChanged()` 内で `HasAnyChanges` を確認し `_hasEditedDuringReview` に累積する。新しいジョブを選択した時点でフラグをリセットし、`CompleteReviewButtonLabel` の変更通知も同時に行う（`_hasEditedDuringReview` は `[ObservableProperty]` でないため手動通知が必要）。

| 判定条件 | 設定される `ReviewStatus` |
|---------|--------------------------|
| どのページも編集していない | `Completed`（完了） |
| いずれかのページを編集した | `CompletedWithEdit`（修正完了） |

ボタン押下後は `ResetAllEdits` を呼ばない（編集状態は維持）。

### 4-3. ステータス変更ルール

- 初期値は `Unreviewed`（既存データの `reviewStatus` フィールド欠落時も同様）
- `Unreviewed` → `Completed` / `CompletedWithEdit`：確認完了ボタンで遷移
- 完了状態から別の完了状態への再確定も可（作業ミス訂正のため）
- `Completed` / `CompletedWithEdit` → `Unreviewed`：本バージョンのスコープ外（§8 参照）

### 4-4. バッジの即時更新

確認完了後は `SelectedHistoryItem.ReviewStatus` を直接更新することでバッジを即時反映する。`ReviewStatus` は `[ObservableProperty]` で宣言されており、`[NotifyPropertyChangedFor(nameof(ReviewStatusKey))]` / `[NotifyPropertyChangedFor(nameof(ReviewStatusDisplay))]` により依存プロパティへの通知が自動的に行われる。永続化は `IJobHistoryRepositoryPort.GetByReportId()` で最新の `JobHistoryItem` を取得し `Update()` で保存する。

```csharp
var current = _jobHistoryRepository.GetByReportId(SelectedHistoryItem!.ReportIdValue);
if (current is null) return;
var updated = current with { ReviewStatus = status };
_jobHistoryRepository.Update(updated);

// IsSelected と同パターンでプロパティを直接更新
SelectedHistoryItem.ReviewStatus = status;

// 確認ステータスフィルタ適用中に一覧・空状態を即時反映
OnPropertyChanged(nameof(FilteredJobHistory));
OnPropertyChanged(nameof(IsFilteredEmpty));
```

### 4-5. OCR 失敗ジョブのバッジ表示

- `JobStatus` が `Succeeded` 以外のジョブには確認ステータスバッジを表示しない
- テキストが存在しないため確認できるものがなく、「未確認」表示は誤解を招くため
- XAML では `StatusDisplay == "succeeded"` の DataTrigger でバッジの `Visibility` を `Collapsed` / `Visible` に切り替える（§6 参照）

## 6. 実装方針（概要）

### ReviewStatus 値オブジェクト

`JobStatus.cs` と同パターンで作成する。

```csharp
public enum ReviewStatusValue { Unreviewed, CompletedWithEdit, Completed }

public sealed record ReviewStatus
{
    public ReviewStatusValue Value { get; }
    public ReviewStatus(ReviewStatusValue value) => Value = value;

    public static ReviewStatus Unreviewed        => new(ReviewStatusValue.Unreviewed);
    public static ReviewStatus CompletedWithEdit => new(ReviewStatusValue.CompletedWithEdit);
    public static ReviewStatus Completed         => new(ReviewStatusValue.Completed);

    // null・空文字・未知値 → Unreviewed（ローカル JSON の破損に備えたフォールバック）
    public static ReviewStatus FromString(string? value)
    {
        if (string.IsNullOrWhiteSpace(value)) return Unreviewed;
        return value.ToLowerInvariant() switch
        {
            "unreviewed"        => Unreviewed,
            "completedwithedit" => CompletedWithEdit,
            "completed"         => Completed,
            _                   => Unreviewed
        };
    }

    // "unreviewed" / "completedwithedit" / "completed" を返す（ComboBox Tag と一致）
    public override string ToString() => Value.ToString().ToLowerInvariant();
}
```

### JobHistoryItem への追加

```csharp
// required は付けない。既存の new JobHistoryItem { ... } 箇所がコンパイルエラーになるため。
public ReviewStatus ReviewStatus { get; init; } = ReviewStatus.Unreviewed;
```

### JobHistoryItemViewModel への追加イメージ

```csharp
// IsSelected と同パターン
[ObservableProperty]
[NotifyPropertyChangedFor(nameof(ReviewStatusDisplay))]
[NotifyPropertyChangedFor(nameof(ReviewStatusKey))]
private ReviewStatus _reviewStatus = ReviewStatus.Unreviewed;

public JobHistoryItemViewModel(JobHistoryItem item)
{
    _item = item;
    _reviewStatus = item.ReviewStatus; // 初期値を _item から設定
}

// DataTrigger・フィルタ比較用（英字小文字）。既存の StatusDisplay と同パターン
public string ReviewStatusKey => ReviewStatus.ToString();

// 日本語表示用
public string ReviewStatusDisplay => ReviewStatus.Value switch
{
    ReviewStatusValue.Unreviewed        => "未確認",
    ReviewStatusValue.CompletedWithEdit => "修正完了",
    ReviewStatusValue.Completed         => "完了",
    _ => "未確認"
};
```

### ResultViewModel への追加イメージ

`StatusFilter` と同パターンで追加する。

```csharp
// 既存フィールドに属性追加
[ObservableProperty]
[NotifyCanExecuteChangedFor(nameof(CompleteReviewCommand))]
private JobHistoryItemViewModel? _selectedHistoryItem;

// 新規追加
[ObservableProperty]
[NotifyPropertyChangedFor(nameof(FilteredJobHistory))]
[NotifyPropertyChangedFor(nameof(IsFilteredEmpty))]
private string? _reviewStatusFilter;

// ジョブ全体の編集有無フラグ（全ページ横断）
private bool _hasEditedDuringReview;

// 過去ページ OR 現在ページの編集有無で判断
public string CompleteReviewButtonLabel =>
    (_hasEditedDuringReview || TextEditing.HasAnyChanges)
        ? "確認完了（修正あり）"
        : "確認完了（修正なし）";

// FilteredJobHistory 内に追加
if (!string.IsNullOrWhiteSpace(ReviewStatusFilter))
    query = query.Where(j => j.ReviewStatusKey == ReviewStatusFilter);

// IsFilteredEmpty に ReviewStatusFilter を追加
public bool IsFilteredEmpty =>
    JobHistory.Count > 0 &&
    !FilteredJobHistory.Any() &&
    (IsDateFilterActive || !string.IsNullOrWhiteSpace(SearchQuery)
     || !string.IsNullOrWhiteSpace(StatusFilter) || !string.IsNullOrWhiteSpace(ReviewStatusFilter));

// コンストラクタ内
TextEditing.PropertyChanged += OnTextEditingPropertyChanged;

// 名前付きメソッド（Dispose() で解除するためラムダは使わない）
private void OnTextEditingPropertyChanged(object? sender, PropertyChangedEventArgs e)
{
    if (e.PropertyName == nameof(TextEditingViewModel.HasAnyChanges))
        OnPropertyChanged(nameof(CompleteReviewButtonLabel));
}

// Dispose() に追加
TextEditing.PropertyChanged -= OnTextEditingPropertyChanged;

// OnPageChanged を修正（ページ切替前に編集有無を累積）
private void OnPageChanged(int pageIndex)
{
    if (TextEditing.HasAnyChanges)
        _hasEditedDuringReview = true;

    OnPropertyChanged(nameof(CurrentPage));
    UpdateBoundingBoxesForCurrentPage();
    UpdateTextBlocksForCurrentPage(); // この中で HasAnyChanges がリセットされる
    ClearSelection();
}

// CompleteReview コマンド
[RelayCommand(CanExecute = nameof(CanCompleteReview))]
private void CompleteReview()
{
    var hasChanges = _hasEditedDuringReview || TextEditing.HasAnyChanges;
    var status = hasChanges
        ? ReviewStatus.CompletedWithEdit
        : ReviewStatus.Completed;

    var current = _jobHistoryRepository.GetByReportId(SelectedHistoryItem!.ReportIdValue);
    if (current is null) return;
    var updated = current with { ReviewStatus = status };
    _jobHistoryRepository.Update(updated);

    // IsSelected と同パターンでプロパティを直接更新
    SelectedHistoryItem.ReviewStatus = status;

    // 確認ステータスフィルタ適用中に一覧・空状態を即時反映
    OnPropertyChanged(nameof(FilteredJobHistory));
    OnPropertyChanged(nameof(IsFilteredEmpty));
}

private bool CanCompleteReview() =>
    SelectedHistoryItem is not null &&
    SelectedHistoryItem.Status.Value == JobStatusValue.Succeeded;

// OnSelectedHistoryItemChanged を修正（フラグリセット後にラベルを通知）
partial void OnSelectedHistoryItemChanged(JobHistoryItemViewModel? value)
{
    _hasEditedDuringReview = false;
    OnPropertyChanged(nameof(CompleteReviewButtonLabel)); // フラグは [ObservableProperty] でないため手動通知
    if (value is not null)
    {
        ImageDisplay.SourceImagePath = value.FilePath;
        _ = LoadResultAsync(value.ReportIdValue, CancellationToken.None);
    }
}
```

## 8. 懸念点・確認事項

| 項目 | 内容 |
|------|------|
| **複数ページ OCR の編集有無** | `_hasEditedDuringReview` により全ページ横断で追跡する。ただしこのフラグはメモリ上のみで管理されるため、アプリ再起動後はリセットされる。再起動後に途中から確認を再開した場合は編集済みページの情報が失われる（将来の課題） |
| **未確認への戻し操作** | 現バージョンのスコープ外。将来バージョンで対応予定 |
